HTML

    • The base element
    • The link element

  • The body element represents the visible content shown to the user.
    • The header element contains information shown at the top of the webpage?
    • The h1 element is the top level header. (H1[most important]-H6[least important]) are avaliable
    • The main element contains the content of the webpage.
    • The section element surrounds(contains) a "section" of the webpage.
    • The ul element contains an unordered list
    • The ol element contains an ordered list

CSS

  • A margin indicates how much space we want arround the outside of an element
  • A padding indicates how munch space we want arround the content inside of an element
  • Multiple elements can share the same css code by seperating them with a ',' before the {}
  • A '.' before a declaration, designates that declaration as a class selector. these classes can be used on an html element by adding the 'class' attribute

git

  • git commands are always proceded with the git keyword
  • The status command checks what branch we are currently on and shows the changes made to the branch?
  • The checkout command changes the current branch
    • The -b option takes a branch name (-b branch-name) and creates a new branch off of the current branch and switches to it
  • The reset command will undo changes made to the git tree? ( ex: git reset --'manner' 'commit designator'
    • the --hard command will go back to a previous commit
    • the --soft command will go back to a previous commit with all changes since that commit showing up as staged changes
    • the HEAD parameter after that will specify how many commits to go back (eg. HEAD^ or HEAD~1 to go back 1 commit or HEAD~4 to go back 4 commits
    • the HEAD parameter can be replaced with the first 7 digits of the commit hash to reset to a specific commit

JavaScript

  • link to basic javascript info https://developer.mozilla.org/en-US/docs/Learn/Getting_started_with_the_web/JavaScript_basics
  • To add javascript to an html file, add the 'script elemnt' before the /body element
  • To specify the java script file to use use the scr attribute of the script element with a relative file location of the '.js' file
  • In java script variables are declared with the let keyword (let myVariable)
    • Variable names must start with an '_' '$' or a letter
    • Variable names can contain the above symbols as well as digits 0-9
    • variable names are case sensitive
  • In javascript use '===' to check if variables are Strictly Equal
  • both '==' and '===' check for equality
  • In javascript use '!==' to check if variables are NOT equal or apply '!' as NOT in front of equality check
  • comments is javascript are denoted by // for a single line or /* */ for multi line comments
  • semi colons are required to split statements on a single line but are not required. (it is good practice to have semi colons at the end of each statement
  • Operators in Javascript are listed here: https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators
  • Open the dev tools with ctl + shift + I or right click inspect to see any ouput data
  • In javascript the let keyword defines a variable to the current block of code
  • In javascript the var keyword defines a variable globaly or to the function scope regardless of block scope
